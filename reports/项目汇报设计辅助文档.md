## 1. 任务画像：我们在解决一个什么样的问题

在深入方法细节之前，先明确任务本身的特征——方法选择永远是为任务服务的。

### 1.1 任务定义

| 维度 | 描述 |
|------|------|
| **任务类型** | 二分类（Binary Classification） |
| **输入** | 路面灰度图像（128×128 及更大尺寸） |
| **输出** | 有裂缝（Positive） / 无裂缝（Negative） |
| **数据集规模** | 40,000 张（正负各 20,000，1:1 均衡） |
| **领域特点** | 裂缝形态多样（横向、纵向、龟裂、细微裂纹），路面纹理复杂（沥青颗粒、污渍、阴影） |

### 1.2 任务的核心挑战

理解任务挑战，才能理解为什么选某些方法：

**挑战一：裂缝的视觉形态高度不统一。** 有的裂缝是一条贯穿全图的直线，有的像蜘蛛网一样分叉，有的只是一条头发丝般的细纹。这意味着不能用简单的规则（如"检测一条直线"）来解决问题——需要一个能学习复杂模式的模型。

**挑战二：路面背景干扰严重。** 沥青本身的颗粒纹理、路面标线、油渍、阴影、光照不均，都可能被误认为裂缝。这要求方法具备从背景噪声中分辨出真正裂缝的能力。

**挑战三：裂缝边缘的精细度。** 裂缝的核心特征是边缘——裂缝与路面的交界处的灰度突变。这个突变可能很微弱（细裂缝），也可能很剧烈（宽裂缝）。方法需要能捕获不同尺度的边缘信息。

**挑战四：类别均衡但误判代价不对等。** 虽然正负样本各半，但"漏判"（把有裂缝判为无裂缝）在实际应用中的代价远高于"误判"（把无裂缝判为有裂缝）。漏判意味着带裂缝的路面被放行，存在安全隐患。因此，召回率（找到真正裂缝的能力）在评估中权重较高。

### 1.3 我们的方法论框架

基于以上任务特征，本项目覆盖机器学习理论的五个核心环节：

```
数据处理 → 模型选择 → 损失衡量 → 参数优化 → 模型验证
```

这五个环节的每一种选择组合，都构成一条完整的训练工作流。后续各节将逐一展开每个环节的可选方案及其原理。

## 2. 模型可选项：为什么这些模型适合裂缝识别

本系统提供三大类共 13 种模型/方法。下面逐一解释每种模型的核心思想、为什么适合本任务，以及使用时的注意事项。

### 2.1 传统监督学习（7 种）

传统监督学习的特点是：需要人工设计特征（即先通过图像处理提取 HOG、LBP 等数值向量），然后让模型在这些特征上学习分类规则。优势是训练快、可解释性强、对计算资源要求低。

---

#### 2.1.1 决策树（Decision Tree）

**一句话原理**：像"猜谜游戏"一样，通过一连串的"是/否"问题逐步缩小可能性，最终给出分类结论。

**工作方式**：从根节点开始，每个节点问一个问题（如"边缘密度大于 0.05 吗？"），根据答案走向左子树或右子树，直到到达叶节点得到最终判断。树的结构由算法自动从数据中"学"出来。

**为什么适合裂缝识别**：
- 裂缝识别任务中有一些直观可解释的规则（如"边缘密度高 → 可能有裂缝"），决策树天然擅长发掘这类规则。
- 树结构可以被完整打印出来，让人清楚地看到模型是如何做判断的——这对理解和验证方法非常有价值。
- 作为所有方法中最简单、最直观的一个，它提供了重要的性能基线：如果某个复杂模型的效果还不如决策树，就说明复杂模型的引入没有带来实质提升。

**局限性**：单棵树容易"背诵"训练数据（过拟合），在未知数据上表现可能下降。这引出了后面的随机森林——用多棵树来弥补这个缺陷。

---

#### 2.1.2 SVM（支持向量机）

**一句话原理**：在数据空间中寻找一条"最宽的分隔带"，将两类样本尽可能干净地分开。

**工作方式**：想象在桌面上撒两种颜色的豆子。SVM 的目标不是随便画一条线把它们分开，而是找一条让两边的豆子离这条线都尽量远的线。"支持向量"就是最靠近分隔线的那些豆子——它们决定了分隔线的位置。当数据本身线性不可分时，SVM 通过"核函数"将数据映射到高维空间，在高维空间中寻找分隔面。

**为什么适合裂缝识别**：
- 我们提取的手工特征维度很高（HOG 就有 3000+ 维，全特征组合可达 4000+ 维）。SVM 在高维空间中表现特别优秀，因为它只关心"支持向量"（最关键的少数样本），维度高了也不容易被噪声淹没。
- 核函数（尤其是 RBF 核）能处理特征与标签之间的非线性关系——裂缝图像的特征组合往往不是简单线性的。
- SVM 的决策边界仅由少数关键样本（支持向量）决定，具有一定的抗过拟合能力。

**局限性**：训练时间随样本量增长较快；模型文件较大（本项目 SVM 模型约 79MB）；对大规模数据不如集成方法高效。

---

#### 2.1.3 朴素贝叶斯（Naive Bayes）

**一句话原理**：基于贝叶斯定理，假设每个特征独立地贡献于分类概率，然后综合所有特征的"意见"做出判断。

**工作方式**：对每个特征，统计它在正类（有裂缝）和负类（无裂缝）中的分布规律。来了一个新样本后，独立计算每个特征"更像是正类还是负类"，然后把所有特征的意见乘起来，得到最终属于每一类的概率。

**为什么适合裂缝识别**：
- 训练速度极快（本项目约 0.9 秒），适合作为快速验证的基线。
- 在高维特征下，虽然"特征独立"假设在现实中不完全成立，但朴素贝叶斯在实践中常常表现得出乎意料地好，是一种"廉价但好用"的概率基线。
- 输出的是"每个样本属于有裂缝的概率"，这个概率值本身就有很好的可解释性。

**局限性**：特征独立性假设在实际中常被违反（HOG 的相邻梯度方向显然不独立），因此精度通常低于其他方法。在本项目中 F1 约为 0.908，是监督方法中最低的，但差距并不大。

---

#### 2.1.4 随机森林（Random Forest）

**一句话原理**：种很多棵决策树，每棵树看到的数据和特征都略有不同，最后投票决定结果——"三个臭皮匠，顶个诸葛亮"。

**工作方式**：从原始训练集中随机抽样（Bootstrap）生成多份不同的子数据集，每份训练一棵决策树，且每棵树分裂时只能从随机选取的部分特征中选择。预测时，所有树各自投票，少数服从多数。

**为什么适合裂缝识别**：
- 图像特征之间存在复杂的相互作用（如边缘密度高 + LBP 纹理粗糙 + GLCM 对比度大 → 很可能是裂缝），单棵树只能捕捉部分交互，随机森林的多树并行机制能更全面地捕获这些组合规律。
- 裂缝图像中有各种噪声（油渍、阴影、纹理变化），随机森林的随机性使其不会被某一类噪声"带偏"——不同的树可能被不同的噪声影响，但投票机制会稀释个别树的错误。
- 自动输出特征重要性排名：哪些特征对判断裂缝最有帮助？特征重要性图能直观回答这个问题。
- 训练速度快（本项目约 6.8 秒），效果强（F1≈0.944），是非常实用的选择。

**局限性**：模型解释性不如单棵决策树；对极少出现的裂缝模式（如训练集中极罕见的裂缝形态）可能不敏感。

---

#### 2.1.5 逻辑回归（Logistic Regression）

**一句话原理**：把特征的加权组合通过 Sigmoid 函数"压缩"为 0 到 1 之间的概率值，超过 0.5 判为有裂缝。

**工作方式**：每个特征有一个权重，所有权重与特征值的加权和再经过 Sigmoid 函数 $\sigma(x) = 1/(1+e^{-x})$ 映射为概率。训练过程就是找到一组最优权重，使预测概率尽可能接近真实标签。

**为什么适合裂缝识别**：
- 模型极度简单，训练只需约 3.2 秒，输出的概率有直接的统计含义（"这个样本有 87% 的概率属于有裂缝"）。
- 作为线性方法的代表，它提供了重要的对标基线：如果逻辑回归的效果和复杂的非线性模型差不多，说明这个任务本身用线性规则就能解决；如果差距大，说明非线性模式确实重要。
- 实际表现出乎意料地好（F1≈0.937），说明手工特征的质量很高，裂缝的很多模式在线性层面就已经可分了。

**局限性**：只能学习线性决策边界，无法自动捕捉特征间的非线性交互。

---

#### 2.1.6 XGBoost

**一句话原理**：一棵树一棵树地"接力"学习——每棵新树专门修正前面所有树犯的错误，逐步逼近最优解。

**工作方式**：与随机森林的"并行投票"不同，XGBoost 的树是"串行补漏"的。第一棵树做出初步判断，第二棵树重点学习第一棵树犯错的样本，第三棵树再修正前两棵的残差……如此迭代。此外，XGBoost 在损失函数中引入二阶导数信息，比传统梯度提升收敛更快、更准。

**为什么适合裂缝识别**：
- 裂缝识别是一个典型的"细节决定成败"的任务——细裂缝和路面纹理的差异可能很微妙。Boosting 的串行修正机制天然适合处理这种"困难样本"，每一轮迭代都在之前容易分错的裂缝上重点发力。
- 正则化机制（L1/L2 正则、子采样）有效防止过拟合，即使特征维度很高也能保持泛化能力。
- 工业界和 Kaggle 竞赛的长期验证表明，XGBoost 在表格型特征（手工提取的数值特征）上的表现极难被超越。本项目 F1≈0.950，是传统方法中的冠军。

**局限性**：训练时间长（本项目约 165 秒）；模型参数量大，不易直观解释。

---

#### 2.1.7 LightGBM

**一句话原理**：XGBoost 的"轻量加速版"——通过直方图算法和按叶子生长的策略，在保持相近精度的前提下大幅提升训练速度。

**工作方式**：核心技术差异在于两点：一是将连续特征离散化为直方图分箱，大幅减少计算量；二是树的生长策略是"Leaf-wise"（每次选择增益最大的叶子分裂），而非 XGBoost 的"Level-wise"（逐层生长）。Leaf-wise 在相同分裂次数下能长出更深的树，捕获更复杂的模式，但需要额外的叶子数控制来防止过拟合。

**为什么适合裂缝识别**：
- 当样本量较大时（如扩展到几万张图像的全量训练），LightGBM 比 XGBoost 快 3-10 倍的训练速度优势体现明显。
- 直方图算法天然带有一定的正则化效果（分箱相当于对特征做了平滑），对图像特征中的微小噪声有抑制作用。
- 本项目 F1≈0.944，与随机森林持平，但训练时间约 96 秒。

**局限性**：对小数据集（如几百个样本）Leaf-wise 策略容易过拟合；参数调优略复杂（多了 num_leaves 等参数）。

---

### 2.2 深度学习：CrackCNN

**一句话原理**：不需要人工设计特征，让神经网络从原始像素开始，逐层自动学习从"边缘→纹理→局部形状→整体裂缝"的层级化视觉概念。

**工作方式**：CrackCNN 包含 4 个卷积块（通道数从 1→32→64→128→256），每个卷积块内部有 2 层 3×3 卷积 + BatchNorm + ReLU + 2×2 最大池化。这种设计使网络逐层扩大"视野"：浅层看到的是局部边缘和纹理（3×3→6×6→12×12→24×24 的感受野），深层看到的是整体的裂缝形态。最后用全局平均池化压缩空间信息，接一个分类头输出两类概率。

**为什么适合裂缝识别**：
- **端到端学习**：传统方法需要"先提特征再分类"两步走，特征提取的好坏直接影响最终结果；CNN 将两步合为一体，特征提取和分类联合优化，避免了手工设计特征的偏差。
- **层级特征学习**：浅层自动学习边缘检测器（类似 Canny），中层学习纹理模式（类似 LBP），深层学习整体形态（裂缝的走向、分叉模式等），每一层的输出都是为分类目标优化的。
- **对光照和噪声的鲁棒性**：BatchNorm 使网络对输入的光照变化和对比度差异有较强的适应能力，数据增强（训练时的随机变换）进一步提升了泛化性。
- **精度显著领先**：本项目 CNN 的 F1 分数达到 0.992-0.998，全面超越所有传统方法（最高 0.950）。这说明在裂缝图像这个任务上，"让网络自己学特征"确实比"人工设计特征"更有效。

**为什么是轻量级设计（~1.17M 参数）**：
- 裂缝识别是二分类任务，比 ImageNet 的 1000 分类简单得多，不需要 ResNet-50 级别的几十 M 参数。
- 全局平均池化替代全连接层，大幅减少参数量——全连接层往往是参数量的主要来源。
- 轻量化设计使模型在 CPU 上也能较快完成推理，降低可视化系统的使用门槛。

**局限性**：需要 GPU 才能高效训练；对样本量的需求高于传统方法（不过本项目 40,000 张已足够）；可解释性不如传统方法（虽然可以通过 Grad-CAM 等工具一定程度可视化网络关注的区域）。

### 2.3 无监督聚类（5 种）

无监督方法的特点是：不使用标签（即不告诉模型哪些图有裂缝、哪些没有），让算法自行在特征空间中寻找数据的自然分组结构。本项目中无监督方法主要用于探索性分析——回答"不借助标签，手工特征本身能否自然地将裂缝图像和正常图像区分开？"

---

#### 2.3.1 K-Means

**一句话原理**：将数据分成 K 个簇，每个样本属于离它最近的簇中心，不断迭代调整簇中心直到稳定。

**为什么选它**：最经典、最直观的聚类方法，作为聚类分析的基准线。如果连 K-Means 都能在特征空间中自然分开裂缝/正常图像，说明手工特征质量极高——但实际上做不到。

---

#### 2.3.2 GMM（高斯混合模型）

**一句话原理**：假设数据由多个高斯分布混合生成，用 EM 算法（期望最大化）估计每个高斯分布的参数，给出每个样本属于每个簇的概率。

**为什么选它**：与 K-Means 的"硬分配"（每个样本严格属于一个簇）不同，GMM 提供"软分配"（输出属于每个簇的概率），这在裂缝检测中更合理——有些模糊图像可能"介于裂缝和正常之间"。

---

#### 2.3.3 DBSCAN

**一句话原理**：从任意一个点出发，如果它周围一定距离（eps）内有不少于 min_samples 个邻居，就把它们连成一个簇；不断扩展直到没有新点加入。无法连入任何簇的点被标记为噪声。

**为什么选它**：裂缝图像在特征空间中可能形成不规则形状的簇（不是球形），DBSCAN 不预设簇的形状，还能自动识别"离群点"（可能是特征特别异常的图像）。但实际中，在本数据集上 DBSCAN 未能找到有效的二簇结构。

---

#### 2.3.4 层次聚类（Agglomerative Clustering）

**一句话原理**：自底向上地不断合并最相似的两个簇，形成一棵树状结构（谱系图），用户可以在任意层次切分得到所需数量的簇。

**为什么选它**：层次结构能揭示数据的多粒度分组。例如，可能先分出裂缝/正常两大类，然后在裂缝内部进一步分出"宽裂缝"和"细裂缝"子类。本项目层次聚类（Ward 链接）是唯一取得相对有意义结果的无监督方法（ARI≈0.15）。

---

#### 2.3.5 谱聚类（Spectral Clustering）

**一句话原理**：将数据建模为图（样本是节点，相似度是边权），利用图的拉普拉斯矩阵的特征向量将数据映射到低维空间，再在低维空间中做聚类。

**为什么选它**：当数据在原始空间中形成复杂的非凸形状时，谱聚类通过图嵌入的方式能发现其他方法找不到的结构。但本项目中效果不佳（ARI≈0），说明特征空间中裂缝/正常两类严重交织。

### 2.4 模型选择小结

```
监督方法 (7 种传统模型 + 6 种 CNN 损失配置 = 13 种):
  传统模型 F1 梯度: XGBoost(0.950) > RF(0.944) ≈ LightGBM(0.944) > SVM(0.937) ≈ LR(0.937) > DT(0.918) > NB(0.908)
  CNN F1 梯度:    Focal(0.998) > LabelSmooth(0.996) ≈ Dice(0.996) > FocalBalanced(0.994) > CE(0.992)

无监督方法 (5种):
  ARI 梯度:       层次聚类(0.15) > KMeans(0.10) ≈ GMM(0.10) > 谱聚类(0.00) > DBSCAN(无效)
```

**核心发现**：CNN 的端到端学习显著优于所有基于手工特征的方法（F1 从 0.95 跃升到 0.998）。无监督方法整体表现差，说明裂缝/正常的区分不是特征空间中的"自然聚类"，必须依赖监督信号。

## 3. 其他方法可选项：数据处理、特征、损失与优化

除了模型选择外，数据处理管线、特征提取方案、损失函数和参数优化策略同样是构成一条完整工作流的关键组件。

### 3.1 数据划分策略

数据划分决定了"用什么数据训练、用什么数据测试"，直接影响评估结果的可信度。

#### 3.1.1 留出法（Hold-out）—— 当前系统唯一实现的划分方式

**原理**：将数据集按固定比例（如 70/30）随机分为训练集和测试集。训练集用于学习模型参数，测试集用于评估模型在"没见过"的数据上的表现。

**为什么是本项目的默认选择**：
- 简单高效，一次划分即可完成实验，适合快速对比多种模型。
- 在 40,000 样本的规模下，70/30 划分意味着 28,000 训练 + 12,000 测试，测试集足够大，评估结果具有统计稳定性。
- 对于 Gradio 交互式系统，留出法的即时性很重要——用户切换参数后能立刻看到结果。

#### 3.1.2 K 折交叉验证

**原理**：将数据分成 K 份，每次用 1 份测试、K-1 份训练，轮换 K 次取平均。每一份数据都既当过训练集也当过测试集。

**为什么需要它**：单次留出划分可能存在"运气成分"——恰好分到了一个容易的测试集，得分偏高。K 折交叉验证通过多次轮换消除这种偶然性，给出更稳健的性能估计。在写论文、出正式报告时，交叉验证的结果比单次留出更有说服力。

#### 3.1.3 分层抽样（Stratified Sampling）

**原理**：在划分训练/测试集时，保持正负样本的比例与原始数据集一致（各 50%）。

**为什么重要**：如果不分层随机划分，极端情况下可能训练集中全是正样本、测试集中全是负样本——模型将完全失效。分层抽样消除了这种风险，确保训练集和测试集的类别分布一致。本项目默认开启此选项。

---

### 3.2 图像预处理

预处理在特征提取或 CNN 输入之前对原始图像进行清洗和增强，目的是提升图像质量，让后续的模型更容易发现裂缝。

#### 3.2.1 CLAHE（自适应直方图均衡化）

**原理**：将图像分成若干小块（默认 8×8），在每个小块内部独立做直方图均衡化——拉伸该块的像素值范围，使暗处变亮、亮处细节更清晰。与全局直方图均衡化的区别在于：CLAHE 是局部的，不会因为图像某一角过亮而压暗其他区域。

**为什么适合裂缝图像**：
- 真实路面照片常因光照角度不同而出现局部过暗或过亮。CLAHE 对每个局部区域独立增强，能有效补偿这种不均匀光照。
- 在阴影区域中的裂缝经过 CLAHE 处理后，与背景的对比度显著提升，更容易被后续特征提取步骤捕获。

#### 3.2.2 高斯滤波

**原理**：用高斯核（一个中心权重高、边缘权重低的模板）在图像上滑动加权平均。效果是平滑图像、消除高频噪声。

**为什么需要它**：相机传感器的随机噪声（尤其在低光条件下）会在图像中产生细小的亮暗点。这些噪点如果位于裂缝边缘附近，可能干扰边缘检测。高斯滤波能有效抑制这类随机噪声。

#### 3.2.3 中值滤波

**原理**：对每个像素，取它周围邻域的中值替代它本身。与高斯滤波不同，中值滤波是非线性的——它直接"替换"异常值，而非"平均"。

**为什么适合裂缝图像**：
- 椒盐噪声（随机出现的纯黑或纯白像素点）在道路图像中可能由灰尘、小石子引起。中值滤波能干净地去除这类噪声，而不会像高斯滤波那样模糊裂缝边缘。
- 在去除噪声的同时，中值滤波对边缘的保留比线性滤波更好——裂缝边缘的梯度信息不会因平滑而削弱。

#### 3.2.4 组合方案（CLAHE + 中值 / CLAHE + 高斯）

**为什么组合使用**：CLAHE 增强对比度，中值/高斯滤波降噪——两者互补。实验表明 CLAHE + 中值滤波的组合对 F1 分数有约 2-5% 的提升。先增强再降噪的顺序是经过验证的：如果先降噪再增强，噪声可能在增强步骤中被重新放大。

---

### 3.3 特征提取（仅传统模型使用）

手工特征是传统机器学习模型的"眼睛"——模型本身不会看图像，只能看到从图像中提取出来的数值向量。特征的好坏直接决定了传统模型的上限。

#### 3.3.1 HOG（方向梯度直方图）

**直观理解**：将图像分成小格子，在每个格子里统计"边缘的方向分布"。比如某个格子里大部分边缘都是水平的 → 这个格子的 HOG 特征中水平方向的数值就大。

**为什么适合裂缝**：裂缝的本质是边缘——路面和裂缝的交界处有明显的灰度变化。HOG 的核心就是捕获"边缘在哪里、边缘朝什么方向"。横向裂缝的 HOG 特征会在垂直方向上有强响应，纵向裂缝则相反。这是最直接对齐裂缝特性的特征。

**维度**：~3000+ 维（取决于图像尺寸和参数），在传统特征中占比最大。

#### 3.3.2 LBP（局部二值模式）

**直观理解**：对每个像素，比较它与周围邻居像素的大小关系，大于邻居的记 1，小于的记 0，形成一串二进制编码。整张图的 LBP 编码分布（直方图）就描述了图像的"纹理指纹"。

**为什么适合裂缝**：正常路面的沥青纹理是均匀粗糙的，而裂缝区域的纹理模式会发生突变。LBP 能敏感地捕获这种纹理变化。裂缝区域的 LBP 直方图与正常路面区域会存在系统性差异。

**维度**：59 维（uniform 模式），非常紧凑。

#### 3.3.3 GLCM（灰度共生矩阵）

**直观理解**：统计"相隔一定距离、沿特定方向的两个像素，灰度值分别为 a 和 b 的概率"。从这些统计中提取对比度、相关性、能量、均匀度等二次统计量。

**为什么适合裂缝**：
- **对比度**：裂缝区域像素值变化剧烈 → GLCM 对比度高；正常路面变化平缓 → GLCM 对比度低。
- **相关性**：正常路面有规律的纹理方向 → GLCM 相关性较高；裂缝打乱了规律 → 相关性下降。
- GLCM 提供了 HOG 和 LBP 不直接捕获的信息——像素灰度值之间的空间共生关系。

**维度**：48 维（3 个距离 × 4 个角度 × 4 个属性）。

#### 3.3.4 边缘密度

**直观理解**：用 Canny 边缘检测算法提取图像中的所有边缘像素，然后计算边缘像素占整张图的比例。一句话："这张图里有多少比例是边缘？"

**为什么适合裂缝**：这是最直观的单维特征。有裂缝的图像 → 边缘多 → 边缘密度高；无裂缝的平滑路面 → 边缘少 → 边缘密度低。单靠这一个特征就能达到一定的分类准确率，说明它的判别力很强。但仅靠它还不够——路面标线、轮胎痕迹等也会产生边缘。

**维度**：1 维，最迷你但最直观的特征。

#### 3.3.5 全特征组合

HOG（边缘方向）+ LBP（纹理模式）+ GLCM（灰度共生）+ 边缘密度（边缘占比）——四种特征从不同角度描述图像，信息互补。实验结论是全特征组合性能最优，说明裂缝识别确实需要多角度的视觉信息。

---

### 3.4 损失函数（仅 CNN）

损失函数决定了 CNN 在训练时"往哪个方向优化"，即模型以什么标准来衡量自己预测得好不好。

#### 3.4.1 交叉熵损失（CrossEntropy Loss）

**公式思想**：$L = -\sum y \log(\hat{y})$。当真实标签 y=1（有裂缝），模型预测概率越接近 1，损失越小；预测概率接近 0，损失急剧增大。

**定位**：分类任务的"标准配置"，所有方法的基线。它平等对待每一个样本——无论这个样本是容易分还是难分，权重一样。

#### 3.4.2 Focal Loss

**公式思想**：$L = -(1-p_t)^\gamma \cdot \log(p_t)$。多了一个 $(1-p_t)^\gamma$ 因子：当 $p_t$（模型对正确答案的预测概率）很大（模型很自信且正确），这个因子接近 0 → 损失被大幅降低；当 $p_t$ 很小（模型判断错误），因子接近 1 → 损失几乎不变。

**直观理解**：交叉熵是"每道题都一样重要"，Focal Loss 是"做错的题重点扣分，做对的题少扣分"。

**为什么适合裂缝识别**：
- 数据集中大部分裂缝图像可能是"明显裂缝"（宽裂缝、高对比度、易识别），小部分是"困难裂缝"（细微裂缝、与背景混在一起）。Focal Loss 让模型把注意力集中在困难裂缝上，而不被大量简单样本"淹没"。
- 本系统提供三种配置：γ=2（中等聚焦）、γ=3（强聚焦）、α=0.5 + γ=2（加上类别权重）。

**关键理解**：本项目数据集是均衡的（正负各 20,000），因此 Focal Loss 的 α（类别平衡权重）默认设为 None，纯粹靠 γ 聚焦困难样本。这与不均衡数据集场景下的使用方式不同——在不均衡场景下需要 α 来给少数类更大的权重。

#### 3.4.3 Label Smoothing Cross Entropy

**公式思想**：将绝对标签从 [0, 1] 软化为 [ε/2, 1-ε]（如 ε=0.1 时变为 [0.05, 0.95]）。即告诉模型"不要 100% 相信任何标签，给其他可能性留一点概率空间"。

**直观理解**：常规训练中模型被要求"有裂缝的概率要么是 0 要么是 1"，这可能导致模型过度自信。Label Smoothing 相当于说"大概 95% 是有裂缝"——留出 5% 的余地让模型保持适度的不确定性，这种不确定性反而提升了泛化能力。

**为什么有效**：在裂缝识别中，标签本身可能存在模糊性（如图像边缘的微小裂纹该不该算裂缝？），Label Smoothing 的"软化"让模型不会对模糊样本做出极端预测，减少过拟合。

#### 3.4.4 Dice Loss

**公式思想**：$L = 1 - \frac{2 \times 重叠}{预测集 + 真实集}$。直接优化"预测结果与真实标签的重叠程度"——类似于直接优化 F1 分数。

**直观理解**：交叉熵关心"每个像素预测对不对"，Dice Loss 关心"整体上裂缝区域有没有被找到"。它更关注整体的一致性，而非逐样本的精确概率。

**为什么有效**：Dice Loss 对正负样本比例的敏感度低于交叉熵，如果测试环境中裂缝和正常的比例发生变化，Dice Loss 训练出的模型可能更稳定。本项目实验结果显示其效果与 Focal Loss 和 Label Smoothing 非常接近（F1≈0.996）。

---

### 3.5 参数优化策略

除了选择模型和损失函数，还要决定"怎么找到最优的模型参数"。本系统提供四种策略，可根据场景灵活选择。

#### 3.5.1 预训练模型（pretrained）

**原理**：直接加载项目中已经训练好并保存的模型（位于 `outputs/models/` 目录），跳过训练过程，直接在新数据上推理或微调。

**适用场景**：
- 快速查看模型效果，不想等待训练（毫秒到秒级加载）。
- 使用默认数据管线（CLAHE+中值 + 全特征 + 70/30 分层划分），预训练模型完全匹配。
- 如果用户只改了数据管线（如换预处理、换特征子集），系统会加载预训练模型架构，在新数据上做一次 `fit()`——比从零训练快得多。

**当前可用的预训练模型**：
- 传统模型 7 个（DT/SVM/NB/RF/LR/XGBoost/LightGBM），含 GridSearchCV 最优参数。
- CNN 6 个（6 种损失函数各一），含 15 轮训练的权重。
- 无监督 4 个（KMeans/GMM/Agglomerative/Spectral），DBSCAN 因未找到有效 2 簇参数组合而未保存。谱聚类虽已保存但效果极差（ARI≈0），主要用于完整性对比。

#### 3.5.2 手动参数（manual）

**原理**：用户在界面上自行设置所有超参数，系统直接使用这些参数训练模型，不做自动搜索。

**适用场景**：用户对某类模型有经验，知道大致的最优参数范围；或者想快速验证一组特定参数的效果。

#### 3.5.3 网格搜索（GridSearchCV）

**原理**：用户指定参数网格（如 max_depth 可以是 [3, 6, 9, 12]），系统遍历网格中的所有参数组合，通过 K 折交叉验证找到最优的一组。

**适用场景**：不确定哪些参数最好，希望系统自动找到最优组合。代价是训练时间与网格大小成正比。

#### 3.5.4 随机搜索（RandomizedSearchCV）

**原理**：从参数空间中随机抽样 n_iter 组参数进行尝试，而非遍历所有组合。

**适用场景**：参数空间太大时，网格搜索耗时过长（如 5 个参数各 5 个取值 = 3125 组组合）。随机搜索在有限预算内通常能找到一个"足够好"的结果。

#### 3.5.5 搜索的优化目标（scoring）

传统监督模型的搜索以用户选择的指标（默认 F1）为优化目标——系统会保留在交叉验证中 F1 最高的那组参数。CNN 和无监督的搜索目标分别固定为验证集损失和轮廓系数。

## 4. 组合全景：用户可以做出哪些选择，每种组合意味着什么

本系统是一个**链路式交互工具**，用户沿"数据处理→模型选择→超参数→损失函数→优化策略"的路径依次做出选择，最终构成一条完整的训练工作流。以下按模型类型列出关键组合及其工作流解读。

### 4.1 组合空间的五个维度

在理解具体组合之前，先了解用户做选择的五个维度：

```
维度1: 数据管线
  ├── 划分方法: holdout (当前唯一)
  ├── 训练集比例: 0.50 ~ 0.90 (默认 0.70)
  ├── 分层抽样: 开/关 (默认开)
  ├── 预处理: none / clahe / gaussian / median / clahe+gaussian / clahe+median
  ├── 特征子集: hog, lbp, glcm, edge_density 的任意组合 (15种)
  └── 样本量: 200 ~ 4000 (默认 2000)

维度2: 模型类型
  ├── 传统监督 (7种): DT / SVM / NB / RF / LR / XGBoost / LightGBM
  ├── 深度学习 (1种): CrackCNN
  └── 无监督 (5种): KMeans / GMM / DBSCAN / Agglomerative / Spectral

维度3: 模型超参数
  ├── 每种模型有自己专属的参数集 (如 DT 的 max_depth, SVM 的 C 和 kernel)
  └── 超参数默认值来自 GridSearchCV 的最优结果

维度4: 损失函数 / 目标函数
  ├── 传统模型: 各自内置 (如 DT 的 gini/entropy, SVM 的 hinge)
  └── CNN: 6 种可选 (CE / Focal×3 / LabelSmoothing / Dice)

维度5: 优化策略 + 验证
  ├── 策略: pretrained / manual / grid_search / random_search
  ├── K 折数: 2~10
  └── 验证方法: holdout / kfold (无监督: internal/external/both)
```

### 4.2 按模型类型的关键组合解读

以下以"典型组合"的形式展示最常见的使用路径。每个组合都标注了适用的模型类型和推荐的优化策略。

---

#### 组合 A：传统模型 + 预训练参数 + 默认数据管线

```
数据: holdout 70/30 + 分层 + CLAHE+中值 + 全特征 + 2000样本
模型: 任意传统模型 (如 随机森林)
超参数: 使用界面默认值 (GridSearchCV 最优)
优化策略: pretrained
验证: holdout
```

**这是什么工作流**：这是最"开箱即用"的路径。用户选择一个模型，系统加载对应预训练的最优参数模型，在当前数据上做一次快速的 fit + predict，即刻输出混淆矩阵、ROC 曲线、PR 曲线、概率分布图和（树模型的）特征重要性图。

**原理解读**：预训练策略让这套组合的运行时间压缩到秒级（随机森林约 7 秒，朴素贝叶斯不到 1 秒）。用户在界面上可以快速切换 DT → SVM → NB → RF → LR → XGBoost → LightGBM，逐一看每个模型的评估结果，建立对"不同模型在这个任务上的表现差异"的直观感受。

**适合场景**：初步探索、演示、教学——快速看到不同模型的性能差异。

---

#### 组合 B：传统模型 + 手动参数 + 自定义数据管线

```
数据: holdout 80/20 + 不分层 + 仅中值滤波 + 仅 HOG+边缘密度 + 1000样本
模型: 决策树
超参数: max_depth=5, min_samples_split=10, criterion=entropy
优化策略: manual
验证: holdout
```

**这是什么工作流**：用户自定义了每一个环节——减少了样本量（1000）、去掉了 CLAHE 和 LBP/GLCM、换了决策树的分裂准则为 entropy 并限制了深度。系统使用这些"纯手动"配置，从头训练模型并输出评估结果。

**原理解读**：这条路径适合"实验探索"——比如测试"只用 HOG 和边缘密度两种特征，决策树能到什么水平？"通过对比这个结果与"全特征 + 大深度"的结果，用户可以直观理解特征数量和模型复杂度对性能的贡献。

**适合场景**：消融实验（Ablation Study）、特征工程探索、理解每个组件对最终性能的贡献。

---

#### 组合 C：传统模型 + GridSearchCV 自动搜索

```
数据: holdout 70/30 + 分层 + CLAHE+中值 + 全特征 + 2000样本
模型: XGBoost
优化策略: grid_search (3折CV, scoring=f1)
验证: kfold 5折
```

**这是什么工作流**：用户不确定 XGBoost 的最优参数是什么，让系统自动做网格搜索。系统遍历预定义的参数网格（n_estimators × max_depth × learning_rate × subsample = 54 种组合），每组参数做 3 折交叉验证，以 F1 为优化目标选出最优参数组合。训练完成后，再对最终模型做 5 折交叉验证给出稳健的性能估计。

**原理解读**：这是"正式实验"级别的工作流。网格搜索确保找到的参数不是偶然的好（而是在多种参数中脱颖而出的），加上最终的 K 折交叉验证，这个结果可以放心地写进论文。代价是训练时间长——54 组 × 3 折 × 每折训练时间 = 大量计算。

**适合场景**：正式实验、寻找最优参数、准备论文数据。

---

#### 组合 D：传统模型 + RandomizedSearchCV

```
数据: holdout 70/30 + 分层 + CLAHE+中值 + 全特征 + 2000样本
模型: LightGBM
优化策略: random_search (n_iter=30, 3折CV)
验证: kfold 5折
```

**这是什么工作流**：与组合 C 类似，但参数搜索采用随机抽样而非穷举遍历。LightGBM 的参数网格有 81 种组合，随机搜索只需尝试 30 组。虽然可能错过全局最优，但在计算预算有限时是实用的折中方案。

**原理解读**：研究表明，随机搜索在参数空间较大时，用远少于网格搜索的预算就能找到接近最优的参数——因为大多数参数对性能的影响并不大，随机的"广泛覆盖"比穷举的"密集采样"更能命中真正重要的参数区域。

**适合场景**：参数空间大、计算预算有限、需要快速获得"够用"的参数。

#### 组合 E：CNN + 预训练权重 + 不同损失函数

```
数据: holdout 70/30 + 分层 + CLAHE+中值 (CNN直接在处理后图像上训练，不需要手工特征)
模型: CrackCNN
损失函数: Focal Loss (γ=2, α=None)
超参数: dropout=0.5, batch_size=64, lr=0.001
优化策略: pretrained
```

**这是什么工作流**：选择 CNN 模型和 Focal Loss，系统加载对应的预训练权重（`crackcnn_focal_gamma2_best.pth`），直接在测试集上推理并输出评估结果。用户可以通过下拉菜单切换 6 种损失函数，观察不同损失函数对最终性能的影响。

**原理解读**：CNN 与传统模型的最大区别在于——用户不需要选择特征（hog/lbp/glcm/edge_density），CNN 从原始像素端到端学习。预训练权重已包含 15 轮训练的成果，加载后即刻可用。6 种损失函数的切换体现了"同一模型架构 + 不同优化目标"的对比——Focal Loss 关注困难样本、Label Smoothing 防止过拟合、Dice Loss 优化整体重叠——尽管最终效果差异不大（F1 0.992-0.998），但各自的训练动态（Loss 曲线形状、收敛速度）值得对比。

**适合场景**：理解深度学习与传统方法的差异、对比不同损失函数的效果。

---

#### 组合 F：CNN + 完整训练 + GridSearchCV + 训练曲线

```
数据: holdout 70/30 + 分层 + 任意预处理
模型: CrackCNN
损失函数: 任意 (如 CrossEntropy)
优化策略: grid_search (lr × dropout × batch_size = 18组 × 10epochs)
```

**这是什么工作流**：不加载预训练权重，而是从零开始搜索最优超参数。18 组参数组合 × 10 轮搜索用 epoch，每组在验证集上评估选出最优参数，然后用最优参数做完整训练（含早停），最终输出训练曲线（Loss 和 Accuracy vs Epoch）。

**原理解读**：CNN 的网格搜索是三维的（学习率、Dropout、Batch Size），共 18 种组合。每组只训练 10 个 epoch（而非完整的 30+），因为搜索的目标是"比较参数"，不需要收敛到最优。选出的最优参数再用于完整训练。这条路径耗时最长（可能数十分钟），但结果是完全定制化的——用户的数据管线和参数选择从头到尾决定了最终的模型。

**适合场景**：完全自主的实验探索、更换数据集后的重新训练。

#### 组合 G：无监督聚类 + 探索性分析

```
数据: holdout 70/30 + 分层 + CLAHE+中值 + 全特征
模型: 层次聚类 (Agglomerative, Ward链接)
聚类数: 2
优化策略: pretrained (或 manual)
评估: internal_external (同时看内部指标和外部指标)
```

**这是什么工作流**：选择一种无监督聚类方法，系统对特征向量做标准化（StandardScaler）后进行聚类。输出包括：PCA 降维散点图（聚类结果 vs 真实标签并排对比）、Silhouette 轮廓图（簇内紧密度可视化），以及内部评估指标（轮廓系数、Davies-Bouldin、Calinski-Harabasz）和外部评估指标（ARI、NMI）。

**原理解读**：无监督方法的评估体系与监督方法完全不同。因为没有标签指导训练，评估重点在于"簇的质量"：
- 内部指标（不看标签）：簇内是否紧密？簇间是否分离？——如果内部指标已经很低，说明数据本身缺乏自然的簇结构。
- 外部指标（对照标签）：聚类结果与真实裂缝标签有多大程度的一致？——ARI 和 NMI 量化了这种一致性。

本项目的核心发现之一正来自这个组合：所有无监督方法的 ARI < 0.16，说明手工特征空间中裂缝/正常图像没有自然的聚类结构。这一"负面结果"本身具有研究价值——它证明了该任务中监督学习是必要的。

**适合场景**：数据探索、验证特征质量、理解"为什么这个任务需要监督学习"。

#### 组合 H：DBSCAN + 自动 eps 调整

```
数据: 同上
模型: DBSCAN
参数: eps=0.5, min_samples=5 (可能触发自动调整)
优化策略: manual
```

**这是什么工作流**：DBSCAN 对参数极其敏感。如果用户设定的 eps 太小，所有点都会被标记为噪声（全 -1）。系统的保护机制会自动计算 k-距离曲线的 75 分位数作为新的 eps 值，并重新聚类。用户可以观察 eps 调整前后聚类结果的变化。

**原理解读**：DBSCAN 的 eps（邻域半径）决定了一个点周围多大范围内的点算"邻居"。eps 太小 → 几乎没有邻居 → 全是噪声；eps 太大 → 所有点都是邻居 → 全是一个簇。在裂缝数据的高维特征空间中，找到一个合适的 eps 使聚类结果为 2 簇且有意义，本身就是一项挑战——本项目未找到这样的参数组合。

**适合场景**：理解密度聚类的工作原理、体会高维空间中"密度"概念的不直观性。

### 6.5 关键技术决策备忘

- **特征提取在划分后进行**（不在全量数据上 fit 任何变换），防止信息从测试集泄漏到训练过程。
- **Focal Loss 的 α 默认为 None**（不启用类别权重），因为数据集 1:1 均衡，不需要对某一类额外加权。
- **GridSearchCV 的 scoring 默认为 F1**（而非 Accuracy），因为 F1 同时考虑精确率和召回率，且裂缝检测任务中召回率的重要性更高。
- **DBSCAN 被排除在预训练模型清单外**——在本数据集的特征空间中未能找到有效的 2 簇参数组合。
- **数据管线结果有内存缓存**（`_DATA_CACHE`）：同一组预处理+特征+划分参数在一次运行中只计算一次，后续调用直接复用，大幅提升多模型对比时的响应速度。
