## 1. 方法总览

本项目围绕路面裂缝图像识别任务，覆盖机器学习理论五环节：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 数据处理  │ →  │ 模型选择  │ →  │ 损失衡量  │ →  │ 参数优化  │ →  │ 模型验证  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
  3维×6×7          7+1+5          6种损失          18组合          5+3指标
  划分/预处理/      监督+CNN       CE/Focal        lr×dropout      混淆矩阵
  特征组合         +无监督        /Smooth/Dice     ×batch          ROC/PR/CV
```

| 环节 | 对比维度 | 配置数量 |
|------|---------|:--:|
| 数据处理 | 划分策略(5) × 预处理(6) × 特征(7) | 系统对比 |
| 模型选择 | 监督 7 种 + CNN 1 种 + 无监督 5 种 | 13 种 |
| 损失函数 | CE / Focal×3 / LabelSmooth / Dice | 6 种 |
| 参数优化 | lr × dropout × batch_size + GridSearchCV | 18+ 组合 |
| 模型验证 | 混淆矩阵 + ROC + PR + 训练曲线 + CV | 5+ 图表 |

## 2. 数据处理

### 2.1 数据集

| 属性 | 值 |
|------|-----|
| 总样本数 | 40,000 张（正负各 20,000） |
| 正样本 (Positive) | 含裂缝的路面图像 |
| 负样本 (Negative) | 无裂缝的正常路面图像 |
| 类别平衡 | ✅ 1:1，天然均衡 |

### 2.2 数据划分策略

| 方法 | 原理 | 选择原因 |
|------|------|---------|
| **留出法 (Hold-out)** | 按固定比例（70/30、80/20、90/10）随机拆分训练/测试集 | 简单高效，适合快速对比实验 |
| **K 折交叉验证** | 分 K 份，轮换 K 次取平均性能 | 评估更稳定，减少单次划分的方差 |
| **分层抽样 (Stratified)** | 划分时保持各类别比例不变 | 确保训练/测试集类别分布一致，避免某类被"抽空" |

**结论**：留出法 70/30 + 分层抽样是计算效率与评估稳定性的最优平衡点。

### 2.3 图像预处理方法

| 方法 | 原理 | 为什么选它 |
|------|------|-----------|
| **CLAHE**（自适应直方图均衡化） | 将图像分块，每块内局部增强对比度 | 路面图像常有光照不均，CLAHE 能局部提升裂缝与背景的对比度 |
| **高斯滤波** | 加权平均卷积，抑制高斯噪声 | 去除传感器引入的随机噪声 |
| **中值滤波** | 取邻域中值，非线性滤波 | 去除椒盐噪声，同时保留裂缝边缘 |
| **CLAHE + 中值** | 先增强对比度，再去除噪声 | 在精度和鲁棒性间取得最佳平衡 |

**实验结论**：CLAHE + 中值滤波的组合管线效果最优（F1 提升约 2-5%）。

### 2.4 特征提取方法

| 特征 | 维度 | 原理 | 为什么选它 |
|------|:--:|------|-----------|
| **HOG**（方向梯度直方图） | ~3000+ | 统计局部梯度方向和强度 | 裂缝具有明显的方向性边缘，HOG 天然适合捕获 |
| **LBP**（局部二值模式） | 59 | 比较像素与邻域，编码为二进制模式 | 描述路面微观纹理，裂缝区域纹理模式异于正常路面 |
| **GLCM**（灰度共生矩阵） | 48 | 统计像素灰度值的空间共生关系 | 裂缝区域的对比度、相关性等统计量与正常路面不同 |
| **边缘密度** | 1 | Canny 边缘像素占比 | 裂缝区域边缘密度显著高于平滑路面，单维判别力强 |

**实验结论**：全特征组合 (HOG+LBP+GLCM+边缘密度) 性能最优。

## 3. 模型选择

### 3.1 监督学习模型（7 种）

| 模型 | 范式 | 核心思想 | 为什么选它 |
|------|------|---------|-----------|
| **决策树 (DT)** | 树模型 | 递归划分特征空间，叶节点输出类别 | 可解释性强，作为基线模型 |
| **SVM** | 核方法 | 寻找最大化类别间隔的超平面，核函数映射到高维 | 高维特征（4000+维）下表现优秀 |
| **朴素贝叶斯 (NB)** | 概率方法 | 贝叶斯定理 + 特征独立性假设 | 训练极快，高维特征下作为概率基线 |
| **随机森林 (RF)** | Bagging 集成 | 多棵决策树 Bootstrap 采样 + 并行投票 | 降方差、抗过拟合、可评估特征重要性 |
| **逻辑回归 (LR)** | 线性方法 | Sigmoid 映射线性组合为概率 | 简单高效、输出概率天然可解释 |
| **XGBoost** | Boosting 集成 | 串行梯度提升 + 二阶泰勒近似 | 精度极高，Kaggle 常胜算法 |
| **LightGBM** | Boosting 集成 | 直方图加速 + Leaf-wise 生长 | 比 XGBoost 快 3-10 倍，内存省 |

### 3.2 深度学习模型（1 种，6 种损失配置）

**CrackCNN** — 自建轻量级 CNN（~1.17M 参数）：

| 设计选择 | 理由 |
|---------|------|
| 4 卷积块 (1→32→64→128→256) | 通道数渐进，从低级纹理到高级语义 |
| BatchNorm + ReLU | 加速收敛、防梯度消失 |
| 全局平均池化 (GAP) | 替代全连接，大幅减参数、防过拟合 |
| Dropout (0.5) | 正则化 |

### 3.3 无监督聚类方法（5 种）

| 方法 | 类别 | 原理 | 为什么选它 |
|------|------|------|-----------|
| **K-Means** | 中心法 | 最小化簇内平方距离 | 经典基线，高效 |
| **GMM**（高斯混合模型） | 概率法 | 多高斯分布混合建模 | 软聚类，输出属于各类的概率 |
| **DBSCAN** | 密度法 | 连通高密度区域成簇 | 识别任意形状簇 + 自动标记噪声点 |
| **层次聚类** | 层次法 | 凝聚/分裂构建树状结构 | 可探索层次关系 |
| **谱聚类** | 图论法 | 图拉普拉斯特征分解 | 处理非凸形状数据 |

## 4. 损失函数选择

> 仅适用于 CNN（传统模型使用各自内置损失：交叉熵/基尼/Hinge 等）

| 损失函数 | 原理 | 适用场景 |
|---------|------|---------|
| **CrossEntropy** | $L = -\sum y \log(\hat{y})$，标准分类损失 | 基线 |
| **Focal Loss (γ=2)** | $L = -(1-p_t)^\gamma \log(p_t)$，抑制易分样本权重 | 关注难分样本，本数据集正负均衡，不启用类别权重 |
| **Focal Loss (γ=3)** | 同上，γ=3 更强的聚焦效应 | 更激进地忽略易分样本 |
| **Focal Loss (α=0.5, γ=2)** | 加上类别权重 α，α=0.5 正负等权 | 显式均衡两类贡献 |
| **Label Smoothing CE** | 将 one-hot 标签软化为 $(1-\varepsilon, \varepsilon)$ | 防止过拟合，提升泛化 |
| **Dice Loss** | $L = 1 - \frac{2|X \cap Y|}{|X| + |Y|}$，直接优化 F1-like 指标 | 适合分割/检测任务，对类别均衡不敏感 |

**注意**：本项目数据集正负样本各 20,000（1:1），Focal Loss 的类别权重 `alpha` 默认为 `None`（不启用），纯粹靠 γ 聚焦难分样本。若数据集不均衡（如裂缝远少于正常路面），应设 `alpha=0.25`（正类少时给正类更小权重）。

## 5. 参数优化

### 5.1 传统模型：GridSearchCV

每种模型独立定义 3-4 维参数网格，3 折交叉验证，以 F1 为优化目标：

| 模型 | 搜索参数 | 搜索空间大小 |
|------|---------|:--:|
| 决策树 | max_depth × criterion × min_samples_split | 6×2×3 = 36 |
| SVM | kernel × C × gamma | 2×3×2 = 12 |
| 朴素贝叶斯 | var_smoothing | 4 |
| 随机森林 | n_estimators × max_depth × min_samples_split | 4×4×3 = 48 |
| 逻辑回归 | C × penalty × solver | 5×2×2 = 20 |
| XGBoost | n_estimators × max_depth × learning_rate × subsample | 3×3×3×2 = 54 |
| LightGBM | n_estimators × max_depth × num_leaves × learning_rate | 3×3×3×3 = 81 |

### 5.2 CNN：三维网格搜索

| 参数 | 搜索范围 | 最佳值 |
|------|---------|:--:|
| 学习率 (lr) | [1e-4, 5e-4, 1e-3] | 1e-3 |
| Dropout | [0.3, 0.5, 0.7] | 0.5 |
| Batch Size | [32, 64] | 64 |

18 种组合 × 15 轮 × 早停(patience=5)，以验证集 Loss 为选择标准。

### 5.3 无监督方法参数搜索

- **K-Means**：K 值 [2..10] 肘部法则 + 轮廓系数
- **GMM**：4 种协方差类型 (full/tied/diag/spherical)
- **DBSCAN**：eps [0.3..3.0] × min_samples [3..20] 网格搜索
- **层次聚类**：4 种链接方式 (ward/complete/average/single)
- **谱聚类**：2 种亲和矩阵 (rbf/nearest_neighbors)

## 6. 评估方法与结果

### 6.1 评估指标体系

| 指标 | 公式 | 含义 | 适用 |
|------|------|------|:--:|
| **准确率 (Accuracy)** | (TP+TN) / All | 整体正确率 | 监督 |
| **精确率 (Precision)** | TP / (TP+FP) | 预测为裂纹中真正是裂纹的比例 | 监督 |
| **召回率 (Recall)** | TP / (TP+FN) | 真实裂纹中被正确识别的比例 | 监督 |
| **F1 分数** | 2×P×R / (P+R) | 精确率与召回率的调和平均 | 监督 |
| **ROC-AUC** | ROC 曲线下面积 | 模型区分正负样本的整体能力 | 监督 |
| **轮廓系数 (Silhouette)** | (b-a)/max(a,b) | 簇内紧密度 vs 簇间分离度 | 无监督 |
| **调整兰德指数 (ARI)** | 聚类与真实标签的一致性 | 校正随机因素后的聚类准确度 | 无监督 |
| **归一化互信息 (NMI)** | 聚类与真实标签共享信息量 | 聚类保留的标签信息量 | 无监督 |

### 6.2 如何看评估指标

**裂缝检测任务的核心关注点**：

- **召回率优先**：漏检裂缝（假阴性）的代价远高于误报（假阳性）——漏检可能导致安全隐患
- **F1 分数**：平衡精确率和召回率，是综合度量
- **ROC-AUC**：>0.9 表示模型有优秀的类别区分能力
- **训练曲线**：训练/验证 Loss 同时下降且间距不扩大 → 未过拟合；验证 Loss 上升而训练 Loss 下降 → 过拟合

### 6.3 监督学习结果（留出法 70/30，CLAHE+中值，全特征）

#### 传统模型在 4000 样本上的结果

| 模型 | 准确率 | F1 分数 | ROC-AUC | 训练时间 |
|------|:--:|:--:|:--:|--:|
| **XGBoost** 🥇 | — | **0.9497** | 0.9816 | 165s |
| **随机森林** 🥈 | — | **0.9438** | 0.9928 | 6.8s |
| **LightGBM** 🥉 | — | **0.9438** | 0.9860 | 96s |
| SVM | — | 0.9371 | 0.9865 | 122s |
| 逻辑回归 | — | 0.9371 | 0.9911 | 3.2s |
| 决策树 | — | 0.9180 | 0.9167 | 18.7s |
| 朴素贝叶斯 | — | 0.9080 | 0.9478 | 0.9s |

> 传统模型 F1 集中在 0.91–0.95，集成方法（XGBoost/RF/LightGBM）明显优于单模型

#### CNN 在 2000 样本上（6 种损失函数，15 轮训练）

| 损失函数 | Val F1 | Test F1 | Test Acc |
|---------|:--:|:--:|:--:|
| **Focal (γ=2)** 🥇 | 1.0000 | **0.998** | 0.998 |

> CNN 全面优于传统方法（F1 0.992–0.998 vs 0.908–0.950），端到端学习避免了手工特征的设计偏差。6 种损失函数表现接近，Focal Loss（γ=2, 无类别权重）略优。

### 6.4 无监督学习结果

| 方法 | ARI | NMI | 轮廓系数 |
|------|:--:|:--:|:--:|
| **层次聚类 (Ward)** 🥇 | **0.1534** | 0.1934 | 0.0628 |
| K-Means | 0.1039 | 0.0808 | 0.0395 |
| GMM | 0.1039 | 0.0808 | 0.0395 |


## 7. 可视化系统设计

### 7.1 系统目标

构建一个基于 **Gradio** 的交互式 Web 界面，允许用户：
- 从下拉菜单/滑块选择数据处理方式、模型类型、损失函数、超参数
- 点击"运行"后系统自动训练/加载模型 → 评估 → 生成图表
- 在同一界面上对比不同模型的结果（ROC 叠加、雷达图、对比表）

### 7.2 5-Tab 布局设计

```
┌─────────────────────────────────────────────────────────┐
│  Tab1: 数据处理  │ Tab2: 模型选择  │ Tab3: 损失函数      │
│  Tab4: 参数优化  │ Tab5: 模型验证                        │
└─────────────────────────────────────────────────────────┘
```

| Tab | 用户可操作 | 系统行为 | 输出 |
|-----|-----------|---------|------|
| **1. 数据处理** | 选划分策略（留出/K折）、比例、预处理方法（CLAHE/高斯/中值…）、特征类型（HOG/LBP/GLCM…） | `data_processing_handler()` → 跑对比实验 | 预处理管线对比图、特征子集对比图、划分策略对比图、最优管线推荐 |
| **2. 模型选择** | 选模型（DT/SVM/NB/RF/LR/XGBoost/LightGBM/CNN/K-Means…）、调超参 | `traditional_model_handler()` / `cnn_handler()` / `unsupervised_handler()` | 混淆矩阵、ROC 曲线、PR 曲线、特征重要性（树模型）、概率分布 |
| **3. 损失函数** | 选损失函数（CE/Focal/LabelSmooth/Dice）、调 Focal 的 α 和 γ | `cnn_handler(loss_fn=…)` → 用选定损失训练 CNN | 训练曲线（Loss + F1 vs Epoch）、测试集指标 |
| **4. 参数优化** | 调学习率、Dropout、Batch Size、Epochs | 网格搜索或直接训练 | 超参数热力图、最优参数组合 |
| **5. 模型验证** | 勾选要对比的模型（多选） | `compare_all_models_handler()` → 统一数据管线，所有模型在相同 test 集上评估 | 多模型对比表、ROC 曲线叠加、PR 曲线叠加、雷达图 |

### 7.4 预训练模型的两层加速策略

| 用户改动 | 系统行为 | 耗时 |
|---------|---------|:--:|
| 什么都不改，只用默认参数看结果 | 加载预训练模型 → `.predict()` 直接推理 | **毫秒级** |
| 改预处理/特征/划分方式 | `prepare_data()` 生成新数据 → 预训练模型在新数据上 `.fit()` | 传统模型**秒级**，CNN **分钟级** |
| 改模型类型/超参数/损失函数 | 加载新模型架构 → GPU 训练 | CNN **分钟级** |
| 多模型对比 | 同一份 `prepare_data()` 数据上逐一评估 | 取决于最慢的模型 |
