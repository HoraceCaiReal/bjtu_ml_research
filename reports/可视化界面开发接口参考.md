## 1. 系统架构概述

### 1.1 整体设计

```
┌─────────────────────────────────────────────────────┐
│                   Gradio 前端界面                      │
│  Tab1:数据处理  Tab2:模型选择  Tab3:损失函数           │
│  Tab4:参数优化  Tab5:模型验证                         │
└──────────────┬──────────────────────────────────────┘
               │ 调用 Python 函数
               ▼
┌─────────────────────────────────────────────────────┐
│              5 个 Handler 回调函数                     │
│  data_processing_handler / traditional_model_handler │
│  cnn_handler / unsupervised_handler                 │
│  compare_all_models_handler                         │
└──────────────┬──────────────────────────────────────┘
               │ 加载预训练模型 / 现场训练
               ▼
┌─────────────────────────────────────────────────────┐
│           outputs/models/  预训练模型存储              │
│  traditional/ (7个)  cnn/ (6个)  unsupervised/ (4个) │
└─────────────────────────────────────────────────────┘
```

### 1.2 核心设计原则

- **每个 Handler 是纯 Python 函数**：输入参数，返回 dict，不依赖 Gradio
- **优先加载预训练模型**：若 `outputs/models/` 下有对应模型，直接加载推理（毫秒级）
- **无预训练模型时自动训练**：首次使用或参数不匹配时，自动 GridSearchCV 训练（秒~分钟级）
- **所有图表返回 `matplotlib.figure.Figure` 对象**：直接用 `gr.Plot()` 渲染

### 1.3 数据流

```
用户选择参数 → Handler 函数
  → 加载数据（max_samples 控制样本量）
  → 预处理（CLAHE/高斯/中值滤波）
  → 特征提取（HOG/LBP/GLCM/边缘密度）
  → 加载预训练模型 或 GridSearchCV 训练
  → 推理评估
  → 生成指标 dict + 图表 Figure
  → 返回给 Gradio 渲染
```

**通用规则**：
- 所有 Handler 第一个返回值 dict，结构固定
- `max_samples` 和 `random_seed` 是公共参数
- 返回 `None` 的图表字段表示该模型不支持（如 SVM 无特征重要性）
- `preprocessing` 和 `features` 参数默认值均为 `["clahe", "median"]` 和全部四种特征

## 5. Tab 1：数据处理接口

### 5.1 函数签名

```python
def data_processing_handler(
    split_method: str = "holdout",
    split_ratio: float = 0.7,
    k_folds: int = 5,
    use_stratify: bool = True,
    preprocessing: list = None,
    features: list = None,
    max_samples: int = 2000,
    random_seed: int = 42,
) -> dict:
```

### 5.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `split_method` | str | `"holdout"` | 划分方法：`"holdout"` 或 `"kfold"` |
| `split_ratio` | float | `0.7` | 留出法训练集比例 (0.5~0.9) |
| `k_folds` | int | `5` | K 折交叉验证的 K 值 (5 或 10) |
| `use_stratify` | bool | `True` | 是否分层抽样 |
| `preprocessing` | list | `["none","clahe","gaussian","median","clahe+median"]` | 预处理方法列表 |
| `features` | list | `["hog","lbp","glcm","edge_density"]` | 特征类型列表 |
| `max_samples` | int | `2000` | 加载样本数上限 |
| `random_seed` | int | `42` | 随机种子 |

### 5.3 返回值

```python
{
    "comparison_table": pd.DataFrame,      # 预处理管线对比表（主表）
    "preprocessing_table": pd.DataFrame,   # 同上
    "feature_table": pd.DataFrame,         # 特征子集对比表
    "split_table": pd.DataFrame,           # 划分策略对比表
    "best_pipeline": str,                  # 最优预处理管线名称
    "best_features": str,                  # 最优特征组合名称
    "best_split": str,                     # 最优划分策略名称
    "preprocessing_plot": plt.Figure,      # 预处理对比横向条形图
    "feature_plot": plt.Figure,            # 特征对比横向条形图
    "split_plot": plt.Figure,              # 划分策略对比柱状图
}
```

### 5.4 在 Gradio 中使用

```python
import gradio as gr

def tab1_render(split_method, split_ratio, k_folds, use_stratify,
                preprocessing, features, max_samples):
    result = data_processing_handler(
        split_method=split_method,
        split_ratio=split_ratio,
        k_folds=k_folds,
        use_stratify=use_stratify,
        preprocessing=preprocessing,
        features=features,
        max_samples=max_samples,
    )
    return (
        result["preprocessing_table"],  # gr.DataFrame
        result["feature_table"],        # gr.DataFrame
        result["split_table"],          # gr.DataFrame
        result["preprocessing_plot"],   # gr.Plot
        result["feature_plot"],         # gr.Plot
        result["split_plot"],           # gr.Plot
        f"**最佳管线**: {result['best_pipeline']}\n"
        f"**最佳特征**: {result['best_features']}\n"
        f"**最佳划分**: {result['best_split']}",
    )
```

## 6. Tab 2：传统模型接口

### 6.1 函数签名

```python
def traditional_model_handler(
    model_type: str = "random_forest",
    preprocessing: list = None,
    features: list = None,
    model_params: dict = None,
    cv_folds: int = 5,
    max_samples: int = 2000,
    random_seed: int = 42,
) -> dict:
```

### 6.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model_type` | str | `"random_forest"` | 7 选 1 |
| `preprocessing` | list | `["clahe","median"]` | 预处理方法 |
| `features` | list | `["hog","lbp","glcm","edge_density"]` | 特征类型 |
| `model_params` | dict | `None` | 自定义参数，`None` 则使用预训练最优参数 |
| `cv_folds` | int | `5` | 交叉验证折数 |
| `max_samples` | int | `2000` | 样本数量上限 |
| `random_seed` | int | `42` | 随机种子 |

**支持的 model_type 值**：
`"decision_tree"` | `"svm"` | `"naive_bayes"` | `"random_forest"` | `"logistic_regression"` | `"xgboost"` | `"lightgbm"`

**model_params 示例**：
```python
# 随机森林
{"n_estimators": 100, "max_depth": 20, "min_samples_split": 5}
# XGBoost
{"n_estimators": 100, "max_depth": 6, "learning_rate": 0.1}
# SVM
{"kernel": "rbf", "C": 1.0}
```

### 6.3 返回值

```python
{
    "model_name": str,                    # 模型类型
    "metrics": {                          # 评估指标
        "accuracy": float,                # 准确率
        "precision": float,               # 精确率
        "recall": float,                  # 召回率
        "f1": float,                      # F1 分数
        "roc_auc": float,                 # ROC-AUC
    },
    "confusion_matrix": plt.Figure,       # 混淆矩阵（含数值标注）
    "roc_curve": plt.Figure,              # ROC 曲线（含 AUC）
    "precision_recall": plt.Figure,       # Precision-Recall 曲线（含 AP）
    "feature_importance": plt.Figure | None,  # 特征重要性（仅树模型，SVM/NB/LR 返回 None）
    "prob_distribution": plt.Figure,      # 预测概率分布直方图
    "cv_results": {"f1_mean": float, "f1_std": float},  # 交叉验证结果
    "best_params": dict,                  # 最优超参数
    "model_path": str | None,             # 预训练模型路径
    "training_time": float,               # 训练/加载耗时（秒）
}
```

### 6.4 图表说明

| 图表字段 | 图类型 | 内容 | 适用模型 |
|----------|--------|------|:--:|
| `confusion_matrix` | 2×2 热力图 | TP/TN/FP/FN + 准确率 | 全部 |
| `roc_curve` | 折线图 | FPR vs TPR + AUC | 全部 |
| `precision_recall` | 折线图 | Recall vs Precision + AP + baseline | 全部 |
| `feature_importance` | 横向条形图 | 特征组重要性（HOG/LBP/GLCM/边缘） | DT/RF/XGBoost/LightGBM |
| `prob_distribution` | 双色直方图 | 正/负类预测概率分布 + 决策阈值线 | 全部 |

### 6.5 在 Gradio 中使用

```python
def tab2_traditional_render(model_type, model_params, max_samples):
    result = traditional_model_handler(
        model_type=model_type,
        model_params=model_params if model_params else None,
        max_samples=max_samples,
    )
    metrics = result["metrics"]
    return (
        f"**{result['model_name']}**\n"
        f"准确率: {metrics['accuracy']:.4f}  |  "
        f"精确率: {metrics['precision']:.4f}\n"
        f"召回率: {metrics['recall']:.4f}  |  "
        f"F1: {metrics['f1']:.4f}  |  "
        f"AUC: {metrics['roc_auc']:.4f}\n"
        f"训练耗时: {result['training_time']:.1f}s",
        result["confusion_matrix"],      # gr.Plot
        result["roc_curve"],             # gr.Plot
        result["precision_recall"],      # gr.Plot
        result["feature_importance"],    # gr.Plot (可能为 None)
        result["prob_distribution"],     # gr.Plot
    )
```

## 7. Tab 3：CNN 接口

### 7.1 函数签名

```python
def cnn_handler(
    preprocessing: list = None,
    loss_fn: str = "cross_entropy",
    focal_alpha: float | None = None,   # None = 无类别平衡（适合均衡数据）
    focal_gamma: float = 2.0,
    label_smoothing_epsilon: float = 0.1,
    optimizer_name: str = "adam",
    learning_rate: float = 1e-3,
    dropout_rate: float = 0.5,
    batch_size: int = 64,
    epochs: int = 30,
    early_stopping_patience: int = 10,
    max_samples: int = 4000,
    random_seed: int = 42,
) -> dict:
```

### 7.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `preprocessing` | list | `["clahe","median"]` | 预处理方法 |
| `loss_fn` | str | `"cross_entropy"` | 损失函数：`"cross_entropy"` / `"focal"` / `"label_smoothing"` / `"dice"` |
| `focal_alpha` | float or None | `None` | Focal Loss 正类权重。`None`=无类别平衡（均衡数据集推荐）；`0.5`=中性权重；仅 `loss_fn="focal"` 时生效 |
| `focal_gamma` | float | `2.0` | Focal Loss γ (0~5)，仅 `loss_fn="focal"` 时生效 |
| `label_smoothing_epsilon` | float | `0.1` | 标签平滑 ε，仅 `loss_fn="label_smoothing"` 时生效 |
| `optimizer_name` | str | `"adam"` | 优化器：`"adam"` / `"sgd"` |
| `learning_rate` | float | `1e-3` | 学习率 |
| `dropout_rate` | float | `0.5` | Dropout 比例 (0~0.9) |
| `batch_size` | int | `64` | 批量大小 |
| `epochs` | int | `30` | 最大训练轮数 |
| `early_stopping_patience` | int | `10` | 早停耐心值 |
| `max_samples` | int | `4000` | 样本上限 |
| `random_seed` | int | `42` | 随机种子 |

### 7.3 返回值

```python
{
    "metrics": {                          # 评估指标（同传统模型）
        "accuracy": float,
        "precision": float,
        "recall": float,
        "f1": float,
        "roc_auc": float,
    },
    "train_history": dict,                # 训练历史数据
    "confusion_matrix": plt.Figure,       # 混淆矩阵
    "roc_curve": plt.Figure,              # ROC 曲线
    "training_curves": plt.Figure,        # 训练 Loss + Accuracy 曲线
    "precision_recall": plt.Figure,       # Precision-Recall 曲线
    "prob_distribution": plt.Figure,      # 预测概率分布
    "best_hyperparams": {                 # 超参数记录
        "loss_fn": str,
        "learning_rate": float,
        "dropout_rate": float,
        "batch_size": int,
        "optimizer": str,
    },
    "model_weights_path": str | None,     # 模型权重路径
}
```

### 7.4 性能提示

- **加载预训练模型**：秒级完成（若 `outputs/models/cnn/` 下有匹配的 `.pth` 文件）
- **现场训练**：根据 `epochs` 和 `max_samples`，需要 1~30 分钟（建议引导用户使用预训练模型）
- **首次使用**：若未预训练，建议在界面显示进度提示

### 7.5 在 Gradio 中使用

```python
def tab3_cnn_render(loss_fn, learning_rate, epochs, max_samples):
    result = cnn_handler(
        loss_fn=loss_fn,
        learning_rate=learning_rate,
        epochs=epochs,
        max_samples=max_samples,
    )
    return (
        f"F1: {result['metrics']['f1']:.4f}  |  AUC: {result['metrics']['roc_auc']:.4f}",
        result["training_curves"],         # gr.Plot
        result["confusion_matrix"],        # gr.Plot
        result["roc_curve"],              # gr.Plot
        result["precision_recall"],        # gr.Plot
        result["prob_distribution"],       # gr.Plot
    )
```

## 8. Tab 4：无监督接口

### 8.1 函数签名

```python
def unsupervised_handler(
    method: str = "kmeans",
    n_clusters: int = 2,
    feature_set: list = None,
    method_params: dict = None,
    max_samples: int = 2000,
    random_seed: int = 42,
) -> dict:
```

### 8.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `method` | str | `"kmeans"` | 聚类方法：`"kmeans"` / `"gmm"` / `"dbscan"` / `"agglomerative"` / `"spectral"` |
| `n_clusters` | int | `2` | 聚类数量（DBSCAN 不使用此参数） |
| `feature_set` | list | `["hog","lbp","glcm","edge_density"]` | 特征子集 |
| `method_params` | dict | `None` | 方法特定参数 |
| `max_samples` | int | `2000` | 样本上限 |
| `random_seed` | int | `42` | 随机种子 |

**method_params 示例**：
```python
# DBSCAN
{"eps": 0.5, "min_samples": 5}
# 层次聚类
{"linkage": "ward"}   # "ward" / "complete" / "average" / "single"
# 谱聚类
{"affinity": "rbf"}   # "rbf" / "nearest_neighbors"
```

### 8.3 返回值

```python
{
    "labels": np.ndarray,                 # 聚类标签（-1 = 噪声点，仅 DBSCAN）
    "internal_metrics": {                 # 内部评估指标
```

## 11. 常见问题

### Q0：多模型对比如何保证在相同数据上评估？

`compare_all_models_handler` 内部通过 `prepare_data()` 统一数据管线 — 只加载一次数据、做一次预处理、一次 train/test 划分，然后所有模型共享同一份 `X_train, X_test`。这避免了每个模型独立加载数据导致的数据不一致问题。

`prepare_data()` 完整定义在 `05_Gradio接口规范.ipynb` §7.5，可独立调用：
```python
data = prepare_data(max_samples=1000, preprocessing=["clahe","median"],
                     features=["hog","lbp","glcm","edge_density"])
X_train, X_test = data["X_train"], data["X_test"]
```

### Q1：Handler 函数在哪？需要导入什么？

Handler 函数定义在各 Notebook 的最后一节（Gradio 接口预留）。每个 Handler 是自包含的 Python 函数，可以直接复制到 `gradio_app.py` 中使用。它们依赖的函数（如 `load_dataset`、`extract_all_features` 等）已在各自 Notebook 中定义，复制时需连同依赖函数一起复制。

### Q2：预训练模型不存在怎么办？

Handler 会自动降级为现场训练：
- 传统模型：GridSearchCV 训练（1~3 分钟）
- CNN：完整训练循环（5~30 分钟）
- 无监督：直接拟合（秒级）

建议首次部署时先在后台运行预训练脚本（`scripts/_pretrain_fast.py`），确保 `outputs/models/` 下有模型文件。

### Q3：如何控制训练速度？

减小 `max_samples` 参数。建议值：
- 快速预览：`max_samples=200`（秒级）
- 正常使用：`max_samples=2000`（分钟级）
- 全量训练：`max_samples=None`（加载全部 40000 张，耗时较长）

### Q4：图表中文乱码怎么办？

Handler 内部已设置 `plt.rcParams["font.sans-serif"] = ["Microsoft YaHei", "SimHei", "DejaVu Sans"]`。如果部署环境无中文字体，图表中文会显示为方块，需安装中文字体包。

### Q7：Focal Loss 的 `focal_alpha` 应该设多少？

- **均衡数据集（正负样本 ≈ 1:1）**：`focal_alpha=None`（默认，推荐）。不按类别加权，纯粹靠 γ 聚焦难样本。本项目的裂缝数据集（20k/20k）属于此类。
- **正类为少数（如 1:10）**：`focal_alpha=0.25`（论文原版默认）。正类权重 0.25，负类权重 0.75。
- **需要中性权重**：`focal_alpha=0.5`。两类等权，但总损失会缩放为原来的 0.5 倍（不影响梯度方向）。

> **早期版本 bug**：旧版 `FocalLoss` 实现将 `alpha=0.25` 均匀施加到所有样本，导致梯度被压缩至 25%，F1 仅 ~0.09。修复后 `alpha=None` 为默认值，在均衡数据上表现正常。
