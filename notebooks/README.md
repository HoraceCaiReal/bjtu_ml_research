# Notebooks 说明

本目录存放研究性专题的全部 Jupyter Notebook，最终提交的**研究报告**也将整合于此。

## 文件命名规范

| 编号 | 文件名 | 用途 | 覆盖的 PDF 环节 |
|------|--------|------|----------|
| 01 | `01_数据处理与特征工程.ipynb` | 共享基础设施：数据加载、划分策略对比、预处理、特征提取 | 数据处理 |
| 02 | `02_传统监督学习对比.ipynb` | 7种分类器对比（DT/SVM/NB/RF/LR/XGBoost/LightGBM） | 模型选择、参数优化、模型验证 |
| 03 | `03_深度学习对比.ipynb` | CNN 完整五环节：架构、损失函数、优化器、超参数搜索 | 数据处理+模型选择+损失衡量+参数优化+模型验证 |
| 04 | `04_无监督学习对比.ipynb` | 5种聚类方法（KMeans/GMM/DBSCAN/层次/谱聚类）对比 | 模型选择、参数优化、模型验证 |
| 05 | `05_Gradio接口规范.ipynb` | 可视化系统统一接口规范文档（留待 Phase 2 实现） | 综合系统设计 |

**设计原则**：
- 每个 Notebook 是自包含的，包含所有需要的代码（数据加载、预处理、模型定义、评估）
- Notebook 之间不相互依赖，可独立运行
- 每个 Notebook 末尾均预留 Gradio 回调函数接口（当前为存根）

## 提交规范

1. **清除输出**：非最终提交前，建议清除 Cell Output 后再提交到 Git（避免仓库膨胀）。
2. **中文字体**：每个 Notebook 开头已内置字体配置代码，无需额外导入外部模块。
3. **代码注释**：所有代码使用中文注释。

## 如何编辑

**主要方式：用 VSCode 直接打开 .ipynb 文件**

确保已激活 conda 环境 `bjtu_ml`，然后用 VSCode 直接打开 `notebooks/` 下的 `.ipynb` 文件即可。VSCode 内置 Jupyter 支持，无需启动额外服务。
