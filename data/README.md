# 数据目录说明

> ⚠️ **本目录被 Git 忽略，绝不上传任何图像文件到仓库！**

## 目录结构要求

请将老师提供的裂纹图像数据集按以下结构放置：

```
data/
├── Negative/          # 20000 张无裂纹图像 (.jpg)
│   ├── 00001.jpg
│   ├── 00002.jpg
│   └── ...
└── Positive/          # 20000 张有裂纹图像 (.jpg)
    ├── 00001.jpg
    ├── 00002.jpg
    └── ...
```

## 如何在代码中使用

项目通过 `.env` 文件配置数据集路径。在 `bjtu_ml_research` 根目录创建 `.env` 文件：

```bash
CRACK_DATA_ROOT=C:/你的实际路径
```

例如，如果你的数据放在 `D:/课程资料/裂纹数据集`：

```bash
CRACK_DATA_ROOT=D:/课程资料/裂纹数据集
```

代码中通过 `src.config.DATA_ROOT` 自动读取该路径。

## 数据保密

- 不向 GitHub 提交任何图像文件
- 不向协作者以外的人分享数据集
- 最终提交时，仅提交代码和报告，不包含数据
