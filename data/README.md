# 数据目录说明

> ⚠️ **本目录被 Git 忽略，绝不上传任何图像文件到仓库！**

## 目录结构要求

请将老师提供的裂纹图像数据集按以下结构放置：

```
data/
├── Negative/          # 20000 张无裂纹图像 (.jpg)
│   ├── 00001.jpg
│   ├── 00002.jpg
│   └── ...
└── Positive/          # 20000 张有裂纹图像 (.jpg)
    ├── 00001.jpg
    ├── 00002.jpg
    └── ...
```

## 如何挂载到 Docker

修改 `docker/docker-compose.yml` 中的数据路径映射：

```yaml
volumes:
  - C:/你的实际路径:/workspace/data:ro
```

例如，如果你的数据放在 `D:/课程资料/裂纹数据集`：

```yaml
  - D:/课程资料/裂纹数据集:/workspace/data:ro
```

`:ro` 表示容器内只读，保护原始数据不被误修改。

## 数据保密

- 不向 GitHub 提交任何图像文件
- 不向协作者以外的人分享数据集
- 最终提交时，仅提交代码和报告，不包含数据
