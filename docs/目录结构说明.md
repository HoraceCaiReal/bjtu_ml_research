## 各目录职责详解

### `_backup/` — 旧版代码备份

- 存放原 `src/` 模块化代码和 `tests/` 测试代码的历史版本
- 仅供参考，**不参与当前开发**

### `data/` — 数据集（本地-only）

- **不上传 Git**，已加入 `.gitignore`
- 协作者各自在本地存放，通过 `.env` 文件配置路径
- `.env` 中设置 `CRACK_DATA_ROOT` 变量指向数据集目录

### `docs/` — 文档与协作记录

- `协作者初始化说明.md`：面向协作者，从零到能跑代码的完整教程
- `日常协作流程.md`：双方共同的 Git / Issues 使用规范
- `分工说明.md`：评分要求的队伍分工说明（5 人分工 + 贡献占比）
- `项目进展里程碑.md`：项目 7 个阶段的任务跟踪与完成记录
- `讨论记录/README.md`：说明为何使用 GitHub Issues 替代本地 markdown 记录

### `notebooks/` — 研究报告与实验（所有代码内联于此）

- **01_数据处理与特征工程.ipynb**：共享基础设施，包含数据加载、划分策略对比（留出法+K折+分层）、预处理方法（CLAHE/高斯/中值滤波）、特征提取（HOG/LBP/GLCM/边缘密度）、预处理与特征组合对比实验
- **02_传统监督学习对比.ipynb**：7 种分类器（DT/SVM/NB/RF/LR/XGBoost/LightGBM），每种独立小节含原理、参数搜索、评估，末尾有七模型综合对比
- **03_深度学习对比.ipynb**：CrackCNN 架构、6 种损失函数对比（CrossEntropy/Focal×3/LabelSmoothing/Dice）、优化器对比（SGD+Momentum/Adam）、三维超参数网格搜索
- **04_无监督学习对比.ipynb**：5 种聚类方法（KMeans/GMM/DBSCAN/层次/谱聚类）的参数搜索与综合对比，PCA 降维分析，CH 分数评估
- **05_Gradio接口规范.ipynb**：可视化系统的统一接口规范文档，含 `prepare_data()` 接口定义
- 每个 Notebook 是**自包含**的，可独立运行

### `src/` — 源代码

- **gradio_app.py**（2974 行）：Gradio 可视化系统的完整实现，自包含无外部模块依赖
  - 图像/特征工具函数（加载、预处理、HOG/LBP/GLCM/边缘密度提取）
  - 数据管线 `prepare_data()`（内存缓存）
  - 模型定义（CrackCNN + 6 种损失函数）
  - Pipeline 引擎（传统/CNN/无监督，支持 pretrained/manual/grid/random）
  - 单图检测（滑窗切块推理，支持大尺寸真实照片）
  - Gradio Blocks UI（3 Tab：模型配置 / 模型评估 / 单图检测）

### `scripts/` — 辅助脚本

- **预训练脚本**：`_pretrain_all.py`、`_pretrain_fast.py` — 批量预训练所有模型
- **批量验证**：`batch_combo_verify.py`、`batch_unsup_verify.py`、`optimized_batch.py` 等
- **图表生成**：`generate_ppt_charts.py` — 生成 PPT 使用的 F0-F17 图表
- **测试**：`e2e_test.py`、`smoke_test*.py` — 端到端测试与冒烟测试
- 日常开发不需要使用

### `reports/` — 报告与图表

- `组合验证评估报告.md`：通过 `run_pipeline()` 运行 30 组方法组合的定量验证报告
- `项目汇报设计辅助文档.md`：13 种模型/方法的方法论解读与组合工作流说明
- `可视化界面开发接口参考.md`：Gradio 系统后端接口文档（5 个 Handler 函数）
- `ppt图表输出/`：PPT 使用的 F0-F17 图表（PNG）、数据 CSV、验证报告

### `outputs/` / `videos/` — 提交物目录

- `outputs/models/`：预训练模型文件（传统 7 个 `.joblib`、CNN 6 个 `.pth`、无监督 4 个 `.joblib`）
- `outputs/results/`：实验结果 CSV
- `outputs/combo_verify/`：组合验证输出（JSON 指标 + PNG 图表）
- `videos/`：5-10 分钟操作演示视频（不上传 Git，单独提交）

## 配置文件说明

### `environment.yml`

Anaconda 环境定义文件，**团队共享**。包含：
- Python 3.10
- PyTorch CUDA 11.8 版本（通过 pip + `--extra-index-url` 安装，无 GPU 时自动回退 CPU）
- NumPy、Pandas、scikit-learn、scikit-image 等（conda 安装更稳定）
- OpenCV（pip 版，避免 conda/pip DLL 冲突）
- XGBoost、LightGBM（梯度提升树对比）
- Gradio >= 4.36.0（可视化系统前端框架）
- Jupyter、ipywidgets 等交互式开发依赖
- `python-dotenv`（读取 `.env`）、`nbstripout`（清理 Notebook 输出）、`ruff`（代码格式化）

### `.env` 与 `.env.example`

- `.env.example`：路径模板，**上传 Git**
- `.env`：真实路径，**不上传 Git**（已加入 `.gitignore`）
- 协作者克隆后执行 `copy .env.example .env`，然后修改为自己的路径

### `pyproject.toml`

Ruff 格式化规则配置：
- 目标 Python 3.10
- 行宽 88（与 Black 兼容）
- 启用 PEP8 规范检查 + 导入排序

### `.gitattributes`

- 配置 `nbstripout` Git 过滤器，自动清除 `.ipynb` 输出
- 设置文本文件使用 LF 换行符（跨平台一致性）

### `requirements.txt`

pip 依赖清单，是 `environment.yml` 的备用版本。包含所有项目实际使用的第三方库，与 `environment.yml` 中的 pip 段保持同步。

## 协作时的文件注意事项

| 操作 | 注意 |
|------|------|
| 修改 `environment.yml` | 必须微信告知对方，双方同步更新环境 |
| 修改 `requirements.txt` | 与 `environment.yml` 的 pip 段保持同步 |
| 修改 `.vscode/settings.json` | 团队共享配置，修改前说一声 |
| 修改 `notebooks/` 下的文件 | 各 Notebook 独立，一般不会冲突 |
| 修改 `src/gradio_app.py` | 主要由王梓铭维护，改前微信沟通 |
| 往 `data/` 放数据集 | 正确做法，但不会被 Git 记录 |
| 往仓库根目录放 `.jpg` / `.pth` | **错误**，会被 `.gitignore` 拦截，但请检查不要强制添加 |
