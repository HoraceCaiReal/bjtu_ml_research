## 各目录职责详解

### `data/` — 数据集（本地-only）

- **不上传 Git**，已加入 `.gitignore`
- 双方各自在本地存放，通过 `.env` 文件配置路径
- 代码中统一使用 `from src.config import DATA_ROOT` 读取，不硬编码路径

### `docs/` — 文档与协作记录

- `协作者初始化说明.md`：面向协作者，从零到能跑代码的完整教程
- `日常协作流程.md`：双方共同的 Git / Issues 使用规范
- `分工说明.md`：评分要求的队伍分工说明，需填写真实姓名和贡献占比
- `讨论记录/README.md`：说明为何使用 GitHub Issues 替代本地 markdown 记录

### `notebooks/` — 研究报告与实验

- **01-03**：按模块拆分的实验 notebook，各自独立开发
- **04**：最终整合的报告 + 综合展示系统，**双方联合维护**
- 日常提交前清除输出（nbstripout 会自动处理）
- 最终提交给老师前可保留关键输出

### `src/` — 可复用的 Python 模块

**设计原则**：把重复的、冗长的代码抽到 `.py` 文件中，Notebook 里只保留调用和展示逻辑。

| 文件 | 职责 | 被谁调用 |
|------|------|----------|
| `config.py` | 读取 `.env`，提供 `DATA_ROOT`、`DEVICE` | 所有模块 |
| `data_utils.py` | 数据加载、划分、预处理 | `01_数据探索`、`02_传统机器学习` |
| `plot_config.py` | 一键设置 matplotlib 微软雅黑字体 | 所有画图 notebook |
| `models/traditional.py` | 决策树、SVM 模型封装 | `02_传统机器学习` |
| `models/unsupervised.py` | K-Means 聚类封装 | `02_传统机器学习` |
| `models/cnn.py` | CNN 网络定义 | `03_深度学习` |
| `training/losses.py` | 损失函数封装 | `02`、`03` |
| `training/optimizers.py` | 参数搜索策略 | `02`、`03` |
| `evaluation/metrics.py` | 评价指标计算 | 所有实验 notebook |

### `reports/` / `videos/` / `outputs/` — 提交物目录

- 保留空目录结构（通过 `.gitkeep`），但内容不上传 Git
- `reports/`：最终导出的 PDF/HTML 报告
- `videos/`：5-10 分钟操作演示视频
- `outputs/`：训练过程中生成的图片、日志等

## 配置文件说明

### `environment.yml`

Anaconda 环境定义文件，**团队共享**。包含：
- Python 3.10
- PyTorch CUDA 11.8 版本（通过 pip + `--extra-index-url` 安装，无 GPU 时自动回退 CPU）
- NumPy、Pandas、scikit-learn 等（conda 安装更稳定）
- OpenCV（pip 版，避免 conda/pip DLL 冲突）
- Jupyter、ipywidgets 等交互式开发依赖
- `python-dotenv`（读取 `.env`）、`nbstripout`（清理 Notebook 输出）、`ruff`（代码格式化）

### `.env` 与 `.env.example`

- `.env.example`：路径模板，**上传 Git**
- `.env`：真实路径，**不上传 Git**（已加入 `.gitignore`）
- 协作者克隆后执行 `copy .env.example .env`，然后修改为自己的路径

### `pyproject.toml`

Ruff 格式化规则配置：
- 目标 Python 3.10
- 行宽 88（与 Black 兼容）
- 启用 PEP8 规范检查 + 导入排序

### `.gitattributes`

- 配置 `nbstripout` Git 过滤器，自动清除 `.ipynb` 输出
- 设置文本文件使用 LF 换行符（跨平台一致性）

## 协作时的文件注意事项

| 操作 | 注意 |
|------|------|
| 修改 `environment.yml` | 必须微信告知对方，双方同步更新环境 |
| 修改 `.vscode/settings.json` | 团队共享配置，修改前说一声 |
| 修改 `src/` 下的通用模块 | 会影响多个 notebook，改前确认无人正在使用 |
| 修改 `notebooks/04_综合展示系统.ipynb` | 双方可能同时改，建议先微信协调 |
| 往 `data/` 放数据集 | 正确做法，但不会被 Git 记录 |
| 往仓库根目录放 `.jpg` / `.pth` | **错误**，会被 `.gitignore` 拦截，但请检查不要强制添加 |
