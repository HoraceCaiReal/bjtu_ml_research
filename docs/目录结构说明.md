## 各目录职责详解

### `_backup/` — 旧版代码备份

- 存放原 `src/` 模块化代码和 `tests/` 测试代码的历史版本
- 仅供参考，**不参与当前开发**

### `data/` — 数据集（本地-only）

- **不上传 Git**，已加入 `.gitignore`
- 协作者各自在本地存放，通过 `.env` 文件配置路径
- `.env` 中设置 `CRACK_DATA_ROOT` 变量指向数据集目录

### `docs/` — 文档与协作记录

- `协作者初始化说明.md`：面向协作者，从零到能跑代码的完整教程
- `日常协作流程.md`：双方共同的 Git / Issues 使用规范
- `分工说明.md`：评分要求的队伍分工说明，需填写真实姓名和贡献占比
- `讨论记录/README.md`：说明为何使用 GitHub Issues 替代本地 markdown 记录

### `notebooks/` — 研究报告与实验（所有代码内联于此）

- **01_数据处理与特征工程.ipynb**：共享基础设施，包含数据加载、划分策略对比（留出法+K折+分层）、预处理方法（CLAHE/高斯/中值滤波）、特征提取（HOG/LBP/GLCM/边缘密度）、预处理与特征组合对比实验
- **02_传统监督学习对比.ipynb**：7 种分类器（DT/SVM/NB/RF/LR/XGBoost/LightGBM），每种独立小节含原理、参数搜索、评估，末尾有七模型综合对比
- **03_深度学习对比.ipynb**：CrackCNN 架构、6 种损失函数对比、优化器对比、三维超参数网格搜索
- **04_无监督学习对比.ipynb**：5 种聚类方法（KMeans/GMM/DBSCAN/层次/谱聚类）的参数搜索与综合对比
- **05_Gradio接口规范.ipynb**：可视化系统（Phase 2）的统一接口规范文档
- 每个 Notebook 是**自包含**的，可独立运行
- 每个 Notebook 末尾预留 Gradio 回调函数接口（当前为存根）

### `scripts/` — 辅助脚本

- 存放 Notebook 构建脚本等辅助工具
- 日常开发不需要使用

### `reports/` / `videos/` / `outputs/` — 提交物目录

- 保留空目录结构（通过 `.gitkeep`），但内容不上传 Git
- `reports/`：最终导出的 PDF/HTML 报告
- `videos/`：5-10 分钟操作演示视频
- `outputs/`：训练过程中生成的图片、日志等

## 配置文件说明

### `environment.yml`

Anaconda 环境定义文件，**团队共享**。包含：
- Python 3.10
- PyTorch CUDA 11.8 版本（通过 pip + `--extra-index-url` 安装，无 GPU 时自动回退 CPU）
- NumPy、Pandas、scikit-learn 等（conda 安装更稳定）
- OpenCV（pip 版，避免 conda/pip DLL 冲突）
- Jupyter、ipywidgets 等交互式开发依赖
- XGBoost 和 LightGBM（梯度提升树对比）
- `python-dotenv`（读取 `.env`）、`nbstripout`（清理 Notebook 输出）、`ruff`（代码格式化）

### `.env` 与 `.env.example`

- `.env.example`：路径模板，**上传 Git**
- `.env`：真实路径，**不上传 Git**（已加入 `.gitignore`）
- 协作者克隆后执行 `copy .env.example .env`，然后修改为自己的路径

### `pyproject.toml`

Ruff 格式化规则配置：
- 目标 Python 3.10
- 行宽 88（与 Black 兼容）
- 启用 PEP8 规范检查 + 导入排序

### `.gitattributes`

- 配置 `nbstripout` Git 过滤器，自动清除 `.ipynb` 输出
- 设置文本文件使用 LF 换行符（跨平台一致性）

## 协作时的文件注意事项

| 操作 | 注意 |
|------|------|
| 修改 `environment.yml` | 必须微信告知对方，双方同步更新环境 |
| 修改 `.vscode/settings.json` | 团队共享配置，修改前说一声 |
| 修改 `notebooks/` 下的文件 | 各 Notebook 独立，一般不会冲突 |
| 往 `data/` 放数据集 | 正确做法，但不会被 Git 记录 |
| 往仓库根目录放 `.jpg` / `.pth` | **错误**，会被 `.gitignore` 拦截，但请检查不要强制添加 |
