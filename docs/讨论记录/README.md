# 讨论记录

> 本目录保留用于存放需要长期归档的文档附件。
> 
> **日常技术讨论已迁移至 GitHub Issues：** [https://github.com/HoraceCaiReal/bjtu_ml_research/issues](https://github.com/HoraceCaiReal/bjtu_ml_research/issues)

## 为什么用 GitHub Issues？

| 优势 | 说明 |
|------|------|
| 自动归档 | 所有讨论永久保存在仓库中，老师可直接查看 |
| 可搜索 | 按标签、关键字快速查找历史讨论 |
| 可引用 | Issue 编号可在提交信息中引用（如 `fix #3`） |
| 多媒体 | 支持直接粘贴截图、上传文件 |

## 讨论分类标签

创建 Issue 时请选择对应标签：

- `讨论` — 方案设计、技术选型等一般性讨论
- `数据` — 数据集处理、路径配置相关问题
- `模型` — 决策树、SVM、CNN、K-Means 等模型问题
- `环境` — Anaconda、依赖安装、环境问题
- `报告` — 最终报告整合、分工说明

## 使用规范

1. **标题清晰**：`[数据] 关于训练集划分比例的讨论` 而不是 `有个问题`
2. **一次一个主题**：不要把数据问题和模型问题混在一起发
3. **讨论完关闭**：结论明确后由发起人点击 Close issue
4. **重要结论同步到代码**：如果讨论产生了配置变更或代码修改，另开 PR 或直接 push 代码，并在提交信息中引用 Issue 编号
